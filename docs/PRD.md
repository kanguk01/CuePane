# CuePane PRD

## 제품 한 줄 정의

이름 붙인 윈도우를 검색하면, 그 윈도우만 또는 그때 함께 보이던 같은 모니터의 작업 문맥까지 복원하는 macOS 유틸리티.

## 문제

macOS는 앱 단위 전환은 편하지만 정확한 창 단위 복귀는 불안정하다. 멀티모니터와 Stage Manager를 함께 쓰면 원하는 창보다 앱 전체가 튀어나오거나, 같이 쓰던 보조 창 문맥이 깨지기 쉽다.

## 목표

- 앱이 아니라 `정확한 작업 창`을 빠르게 다시 찾는다.
- 이름 붙인 창 하나로 같은 모니터의 작업 문맥을 대표하게 만든다.
- 검색 결과에서 `문맥 전체`, `창만`, `여기로 가져오기`를 바로 실행한다.

## 비목표

- 브라우저 탭 단위 식별
- Spaces / Stage Manager 완전 제어
- 닫힌 창 재생성
- 자동 배치 규칙
- 클라우드 동기화

## 주요 사용자

- 멀티모니터 환경에서 창을 많이 띄우는 개발자
- Slack, Terminal, 브라우저, IDE를 섞어 쓰는 사람
- 앱 전환보다 작업 문맥 복원이 중요한 사용자

## 핵심 시나리오

1. 사용자는 터미널 창을 열고 서버 로그를 본다.
2. 같은 모니터에 Slack이 함께 떠 있다.
3. 터미널 창에 `서버로그`라는 이름을 붙인다.
4. CuePane는 터미널을 앵커로 저장하고 같은 모니터의 visible windows를 문맥으로 기록한다.
5. 나중에 사용자가 `서버로그`를 검색하면, 터미널과 Slack이 다시 함께 앞으로 온다.

## 기능 요구사항

### 필수

- 현재 활성 윈도우에 이름 붙이기
- 같은 모니터 문맥 저장
- 이름, 앱명, 창 제목 기준 검색
- 문맥 전체 복원
- 앵커 창만 복원
- 현재 디스플레이로 문맥 가져오기
- 문맥 업데이트
- 앵커 삭제
- 최근 사용 정렬

### 선택

- 검색 결과 라이브 상태 표시
- 부분 복원 안내
- 최근 앵커 바로가기

## UX 원칙

- 입력은 빠르게, 화면은 작게
- 기본 동작은 `문맥 복원`
- 실패해도 이유가 드러나야 한다
- 수동 그룹 생성 없이 `이름 붙인 창`만으로 사용 가능해야 한다

## 성공 기준

- 검색 오버레이가 150ms 안에 떠야 한다
- 저장된 앵커를 2~3키 입력으로 찾을 수 있어야 한다
- Terminal, Slack, Chrome/Arc, Comet 계열 앱에서 실사용 가능한 복원 성공률을 보여야 한다

## 리스크

- 앱별 AX 윈도우 제어 품질 차이
- Stage Manager 동작 완전 제어 불가
- 창 제목 변화가 심한 앱의 매칭 불안정성

## 메시지

`CuePane는 창을 찾는 앱이 아니라, 작업 문맥을 다시 부르는 앱이다.`
